WHAT THIS SCRIPT DOES
----------------------
Automates the full XRD → crystal structure identification pipeline for your
Fe-SMA project. Three modes:

  1. IMAGE mode   — give it one of your XRD images (JPG/PNG/TIF), it auto-
                    detects peaks and runs them through DiffractGPT.
  2. MANUAL mode  — give it a CSV of (2theta, intensity) data directly.
  3. DEMO mode    — runs with built-in synthetic Fe-Mn-Al-Si-Ni-C peaks so
                    you can test without any files.

When an AGAPI key is set, the script calls the cloud DiffractGPT model at
atomgpt.org and gets back a POSCAR file (predicted crystal structure).
Without a key, it falls back to a local JARVIS-DFT cosine-similarity matcher.

WHAT THE OUTPUT MEANS
---------------------
DiffractGPT output (cloud mode):
  - Returns a POSCAR file saved as "diffractgpt_predicted.POSCAR"
  - POSCAR is a VASP crystal structure format:
      Line 1   : system name
      Line 2   : universal scaling factor (usually 1.0)
      Lines 3-5: lattice vectors (a, b, c) in Angstroms
      Line 6   : element symbols
      Line 7   : number of each element
      Line 8   : "direct" (fractional coordinates) or "cartesian"
      Lines 9+ : atomic positions

  Example output for your alloy (demo):
      Fe2MnAlSiNiC — L2₁ Heusler-type structure
      a ≈ 4.24 Å  (FCC-like lattice, Fm-3m symmetry)
      Fe @ (0,0,0) and (0.25,0.25,0.25)
      Mn @ (0.75,0.75,0.75)
      Al @ (0.5,0.5,0.5)
  → This is the target γ-phase austenite Heusler, consistent with your
    synchrotron XRD patterns.

JARVIS local match output:
  - Ranks top-10 Fe-Al compounds from JARVIS-DFT by cosine similarity
  - Score of 1.0 = perfect match, 0.0 = no match
  - Also shows formation energy (eV/atom) — more negative = more stable
  - Top match for demo peaks: AlFe2Mo (Fm-3m, score 0.82) — FCC-type ✓


IMAGE MODE NOTES
----------------
Your synchrotron XRD images (from Shanghai Synchrotron Radiation Facility)
are 2D diffraction images. The script handles them by:
  1. Converting to grayscale
  2. Averaging a horizontal band of pixels (±5% of image height around --row)
  3. Treating the horizontal pixel axis as a 2θ axis (mapped 10°→90°)
  4. Running scipy peak detection with prominence filtering

Important: this works best on already-integrated 1D XRD line plots or images
where the diffraction pattern is displayed as a horizontal intensity curve.
If your images are 2D Debye-Scherrer ring patterns, you need to first
integrate them (e.g. with PyFAI: pip install pyFAI) before passing to
this script.

To adjust peak sensitivity, use --prominence (default 0.05):
  Lower value  → more peaks detected (noisier)
  Higher value → fewer peaks, only strong ones

The detected-peak plot is saved as "xrd_peaks_detected.png" automatically.

WHAT DiffractGPT IS
-------------------
DiffractGPT (DGPT) is a generative pretrained transformer trained on the
JARVIS-DFT database (80,000+ crystal structures + simulated XRD patterns).
It solves the "inverse problem": given an XRD pattern → predict structure.

Paper: "DiffractGPT: Atomic Structure Determination from X-ray Diffraction
        Patterns Using a Generative Pretrained Transformer"
        J. Phys. Chem. Lett. 2024 — Kamal Choudhary (NIST/JHU)
DOI  : https://pubs.acs.org/doi/10.1021/acs.jpclett.4c03137

Three prediction scenarios (best → worst accuracy):
  1. Formula + XRD peaks   ← what this script does
  2. Element list + peaks
  3. Peaks only

For your project, scenario 1 is ideal since you know the nominal composition.

TROUBLESHOOTING
---------------
Problem : UnicodeEncodeError on Windows
Fix     : Run with PYTHONUTF8=1 prefix:
            PYTHONUTF8=1 python fe_sma_diffractgpt.py demo
          Or in PowerShell:
            $env:PYTHONUTF8=1; python fe_sma_diffractgpt.py demo

Problem : DiffractGPT returns no structure
Fix     : Check AGAPI_KEY is set correctly. Test with:
            python -c "import os; print(os.environ.get('AGAPI_KEY','NOT SET'))"

Problem : JARVIS database download slow
Fix     : It only downloads once (~41 MB) and caches at:
            C:\Users\frank\.jarvis\  or  C:\Users\frank\AppData\Local\jarvis\
          Subsequent runs load from cache (fast).

Problem : Image mode detects too many / too few peaks
Fix     : Adjust --prominence (default 0.05)
            --prominence 0.02  → more sensitive
            --prominence 0.15  → only major peaks
          Also try --row to sample a different horizontal strip of the image.
